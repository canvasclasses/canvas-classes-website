Fixed by appending a per-upload `uuidv4().slice(0,8)` token to the filename, guaranteeing a unique key for every upload regardless of block id or filename truncation — this also prevents a new upload from ever silently overwriting an existing asset still referenced elsewhere (content-safety, §0.6 spirit).
