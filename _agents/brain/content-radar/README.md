# Content Radar (what-to-record briefs)

The [Content Radar](../../workflows/CONTENT_RADAR.md) writes dated briefs here: `YYYY-MM-DD.md`, with the newest copied to `latest.md`. The [founder advisor](../../workflows/FOUNDER_ADVISOR.md) reads `latest.md` so "what to record/build next" is content-aware.

This folder is **institutional** (part of the Canvas Brain), not the founder's personal GBrain. It holds YouTube content-planning decisions, evidence-backed.

It is the **tactical** sibling of [`industry/`](../industry/): `industry/` = "what's the market doing" (strategic); this = "what do I record this week" (tactical). The radar may use the scout's latest scan as one input.

## Each weekly brief follows this shape

```markdown
---
type: content-radar
date: YYYY-MM-DD
cadence: weekly        # or: daily
sources: [product-data, vidiq, web]   # list the ones actually hit this run
---

# Content Radar — YYYY-MM-DD  (week of <exam-season window>)

## 🔁 Last week's calls (close-the-loop)
- <recommendation> → acted on? <yes/no> → performance vs expectation → what it teaches.

## 🎯 Record this week (ranked)
> Rubric: demand × timing-fit × our-students'-need × low-competition. <note any weighting>

### 🎬 #1 — <topic, one line>
- **Suggested title:** "<title>" — vidiq title score: <score>
- **Format:** long-form | Short/Reel | live
- **Why now:** <seasonality reason>
- **Evidence:** <product-data hotspot / comment count / keyword volume + competition / competitor breakout — with numbers/URLs>
- **Serves:** <which student signal>
- **Thumbnail concept:** <one line>

### 🎬 #2 — …
(3–5 cards; fewer if evidence is thin — do not pad)

## 👀 Watchlist (signal seen, not yet actionable)
- <thing to keep watching> — evidence

## 🗂️ Sources hit this run
- <which of the 10 sources were reachable; note any that weren't>
```

## Daily-pass file
The light daily pass appends a short entry to that day's file (or writes one): either a single time-sensitive recommendation (breaking exam news / a competitor breakout) or a one-line "nothing urgent." No padding.

Keep every line **evidence-backed** (a metric, a URL, a count, a product-data number) — same grounding rule as the rest of the brain. No speculation presented as fact.
