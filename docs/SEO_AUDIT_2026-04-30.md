### 1.1 Conflicting `robots.txt` (dynamic robots route is shadowed) — ✅ DONE (PR G `941f239`)

- `app/robots.ts:10` → `https://www.canvasclasses.in/sitemap.xml` ✅
- `public/robots.txt:9` → `https://canvas-classes-website.vercel.app/sitemap.xml` ❌

## 2. WARNINGS — Should fix this month

### 2.1 No `Article` / `BlogPosting` schema on blog posts
`app/blog/[slug]/page.tsx` has metadata but emits no JSON-LD. Add `BlogPosting` with `headline`, `image`, `author` (Person), `datePublished`, `dateModified`, `publisher`.

### 2.2 No `VideoObject` schema despite hundreds of YouTube embeds
`detailed-lectures`, `neet-crash-course`, `2-minute-chemistry`, `quick-recap`, `periodic-trends` all embed YouTube. None emit `VideoObject`. This is the difference between getting a video-thumbnail rich result for "[chapter] video lecture" and not.

### 2.3 Missing `Course` / `LearningResource` schema on chapter landings
Top-level Course schema lives in `app/layout.tsx:163`, but per-chapter pages (`/detailed-lectures/[slug]`, `/the-crucible/[chapterId]`) should each emit their own with `educationalLevel`, `teaches`, `inLanguage: en-IN`, `provider: Canvas Classes`.

### 2.4 Sitemap is missing several indexable URL clusters — ✅ DONE (commit `b133ebb` + NCERT in `99fe5eb`)
`app/sitemap.ts` does not include:

- `/jee-pyqs` and `/jee-pyqs/[chapter]` (top-500 PYQs — none in sitemap)
- `/detailed-lectures/[slug]` (every chapter video page)
- `/ncert-solutions/[classNum]/[chapter]` (every NCERT chapter)
- `/blog` and `/blog/[slug]` (entire blog)
- `/about`, `/2-minute-chemistry`, `/organic-chemistry-hub`, `/chemihex`, `/inorganic-chemistry-hub`, `/physical-chemistry-hub`

That is dozens of indexable pages discoverable only by crawl.

### 2.5 Generic homepage description; thin keyword density — ✅ DONE (bundled in `7a9a16a`)
`app/page.tsx` has its own `metadata` that nearly duplicates `app/components/landing/LandingPage.tsx`. Consolidate to one and front-load: `"JEE & NEET Chemistry Free Coaching by Paaras Sir | Canvas Classes"`.

### 2.6 Crucible question slug titles hurt CTR
`app/the-crucible/q/[slug]/page.tsx:50` produces titles like `ATOM-042: Calculate the number of moles of oxygen... | JEE Main 2024 | Canvas Classes`. The leading `display_id` consumes 8 chars of the meta-snippet that Google clips. Lead with the question, append the ID.

### 2.7 Two question systems = potential duplicate content
Both `/chemistry-questions/[chapter]/[slug]` (legacy V1, force-dynamic) and `/the-crucible/q/[slug]` (V2) exist. If both index and serve overlapping content, Google splits signals. Either canonicalize V1 → V2 (`alternates.canonical` pointing the V1 slug at the V2 UUID URL) or 301-redirect V1 → V2 in `next.config.ts`.

### 2.8 Image alts
Empty alts in `app/salt-analysis/SaltAnalysisHero.tsx` (public) and `app/crucible/admin/blog/BlogAdminClient.tsx` (admin — fine). Fix the salt-analysis one.

### 2.9 Missing `alternates.canonical` on many pages
Pages with metadata but no canonical: `/jee-pyqs`, `/handwritten-notes`, `/ncert-solutions` (uses relative — should be absolute), `/about`, `/interactive-periodic-table`, `/chemistry-questions`, `/detailed-lectures`. Without canonicals, query-string and trailing-slash variants index separately.

### 2.10 Crucible question pages — `generateStaticParams` returns `[]`
`app/the-crucible/q/[slug]/page.tsx:18` returns empty + `revalidate: 86400`. No pages pre-rendered at build, so the first user (often Googlebot) waits for a cold MongoDB query. Pre-render the top 200–500 highest-traffic question slugs.

## 3. OPPORTUNITIES — AI / GEO / LLM ranking

What gets the site cited by ChatGPT, Claude, Gemini, Perplexity. Currently doing **none** of this.

### 3.1 Add `/llms.txt` and `/llms-full.txt`
Anthropic / Mintlify / others read `/llms.txt` as the AI equivalent of robots.txt. Add `public/llms.txt` listing top URLs grouped by topic, and `public/llms-full.txt` with the full content of highest-value pages. Cursor / Claude / ChatGPT will train and cite from it.

### 3.2 Restructure top pages for "answer-first" extraction
LLMs cite the first 100 words. Every chapter / topic page should open with a 50–80-word definitive paragraph:
> *"Mole concept is the SI unit of amount of substance, equal to 6.022×10²³ particles. For JEE Main, the three high-yield formulas are…"*

Then the existing content. Right now pages open with hero animations and CTAs — invisible to AI.

### 3.3 Add `Author` Person schema to every educational page
LLMs weight `author.expertise`, `author.alumniOf`, `author.teaches`. The `Person` schema for Paaras Sir currently lives only in `app/layout.tsx`. Embed it on every chapter / lecture / blog page so "Who teaches NEET chemistry?" has citation surface area.

### 3.4 Build canonical Q&A pages targeting AI queries
Static long-form pages for queries Indian students ask AI assistants:

- `/jee-chemistry-strategy`
- `/neet-chemistry-syllabus-2026`
- `/how-to-study-organic-chemistry-jee`
- `/bitsat-vs-jee-chemistry-difference`
- `/best-chemistry-teacher-for-jee-india`

Each: H1 = the literal query, 1500–2500 words, `FAQPage` + `Article` schema, internal links to tools. These rank for Google featured snippets AND get cited by Perplexity / ChatGPT.

### 3.5 Add `HowTo`, `Quiz`, `SoftwareApplication` schema where applicable
- `/salt-analysis` — `HowTo` for the qualitative analysis procedure
- `/the-crucible` — `Quiz` with sample questions
- `/solubility-product-ksp-calculator` — `SoftwareApplication`

### 3.6 Internal linking — chapter-to-chapter and tool-to-content
Footer links major sections, but **chapter pages don't link laterally**. From `/the-crucible/[chapterId]` link to: the matching flashcard chapter, the detailed-lectures chapter, the NCERT solutions chapter, related JEE PYQs. Google and LLMs use these to understand topical authority clusters.

### 3.7 Add `hreflang` for India targeting
Add `<link rel="alternate" hreflang="en-IN" href="..." />` site-wide. Currently `openGraph.locale: 'en_IN'` is set but no hreflang — explicit signal to Google that the site is for India.

### 3.8 Sentry source-map upload hygiene
`next.config.ts:205` — `withSentryConfig` is configured. Verify in production response headers that source maps are not publicly served (SEO + security concern).
